```yaml

# -- Subchart operator toggles
#
# cloudnative-pg.enabled: install the CNPG operator as a bundled subchart.
#   Independent of postgres.managed — postgres.managed controls whether a CNPG Cluster CR
#   is rendered; cloudnative-pg.enabled controls whether the CNPG operator itself is bundled.
#   Set to false when the CNPG operator is already installed in the cluster (pre-installed mode).
#   Installs into the same namespace as this release (no namespace override). (D-01, D-02)
#
cloudnative-pg:
  enabled: true   # -- default: true (full umbrella install bundles the operator)

# -- cert-manager.enabled: install cert-manager as a bundled subchart.
#   Set to false when cert-manager is already installed in the cluster (pre-installed mode).
#   Installs into the same namespace as this release (no namespace override). (D-01)
#
cert-manager:
  enabled: true   # -- default: true (full umbrella install bundles the operator)
```

Key constraints (per D-02, D-05):
- Both defaults are `true` — aligns with core value ("full stack in a single helm install")
- `cloudnative-pg.enabled` and `postgres.managed` are SEPARATE keys controlling different things:
  `postgres.managed` = whether to render a CNPG Cluster CR; `cloudnative-pg.enabled` = whether to bundle the operator
- Comment must explain this distinction clearly to avoid operator confusion
- Do NOT rename or modify the existing `postgres.managed` key

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| values.yaml → Chart rendering | User-supplied values control which subcharts are installed; boolean type enforcement prevents injection |
| Helm registry → charts/ tarballs | Chart tarballs fetched from external registries over HTTPS; pinned versions prevent unexpected upgrades |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | Chart.yaml version strings | mitigate | Pin exact patch versions (D-04) — `"0.28.0"` and `"v1.20.2"` prevent silent drift to newer operator versions that may be incompatible |
| T-06-02 | Elevation of Privilege | cloudnative-pg operator subchart | accept | CNPG operator requires cluster-wide CRD permissions by design; same risk exists whether pre-installed or bundled; Helm RBAC is operator's responsibility |
| T-06-03 | Spoofing | Helm chart repository URLs | mitigate | Use canonical HTTPS repo URLs (`https://cloudnative-pg.github.io/charts`, `https://charts.jetstack.io`); Chart.lock SHA256 digest (produced in Plan 03) pins content after first fetch |
| T-06-04 | Denial of Service | cert-manager subchart namespace conflict | mitigate | D-01 (same namespace as release) is intentional; operators function cluster-wide; document that pre-installed-operator mode (`enabled: false`) avoids any namespace collision |
</threat_model>
