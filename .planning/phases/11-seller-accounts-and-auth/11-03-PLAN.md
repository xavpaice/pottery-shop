## Tasks

### Task 1: Approval email in Register handler
Read `internal/handlers/auth.go` and `cmd/server/main.go` to understand the existing SMTP pattern (used for order placement emails).

After `SellerStore.Create` succeeds in `AuthHandler.Register`, call a helper:

```go
func sendApprovalEmail(smtpConfig SMTPConfig, adminEmail, approvalToken, baseURL string) error
```

The email body contains a plain-text message and the URL:
`{baseURL}/admin/sellers/approve?token={approvalToken}`

If SMTP is not configured or the send fails, log the error and continue — account creation is not rolled back.

Files:
- `internal/handlers/auth.go` — add sendApprovalEmail helper, call from Register

### Task 2: GET /admin/sellers/approve?token=X handler
Add `ApproveSellerByToken(w, r)` to `AuthHandler` (or a new AdminSellerHandler — keep it on AuthHandler for now to avoid a new file).

Logic:
1. Read `token` query param; if empty return 400
2. Call `SellerStore.GetByApprovalToken(ctx, token)` — if nil return 404
3. Call `SellerStore.Approve(ctx, token)` — sets is_active=true, approval_token=''
4. Render a plain success page: "Seller {name} approved."

Register route in `main.go`: `GET /admin/sellers/approve` — this route does NOT require Basic Auth (the token is the auth). Add a note in code that token expiry is a future enhancement.

Files:
- `internal/handlers/auth.go` — add ApproveSellerByToken method
- `cmd/server/main.go` — register GET /admin/sellers/approve

### Task 3: Startup bootstrap — create admin seller from env vars
Read `cmd/server/main.go`. After the DB pool and SellerStore are initialised, add:

```go
exists, err := sellerStore.AdminExists(ctx)
if err != nil { log.Fatalf(...) }
if !exists {
    adminUser := os.Getenv("ADMIN_USER")
    adminPass := os.Getenv("ADMIN_PASS")
    if adminUser != "" && adminPass != "" {
        if err := sellerStore.CreateAdmin(ctx, adminUser, adminPass); err != nil {
            log.Fatalf("bootstrap admin seller: %v", err)
        }
        log.Printf("bootstrap: created admin seller %s", adminUser)
    }
}
```

Add `CreateAdmin(ctx, email, plaintextPassword string) error` to `SellerStore` in `internal/models/seller.go` — same as `Create` but sets `is_active=true`, `is_admin=true`, and leaves `approval_token=''`.

Files:
- `cmd/server/main.go` — add bootstrap block after DB init
- `internal/models/seller.go` — add CreateAdmin method
