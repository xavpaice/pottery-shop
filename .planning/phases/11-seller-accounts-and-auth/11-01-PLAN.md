### Task 1: Goose migration — sellers table
Create `db/migrations/00002_add_sellers.sql` as a Goose migration.

Files:
- `db/migrations/00002_add_sellers.sql` — new file; Goose up/down with sellers DDL

The migration must use the exact schema from the spec:
```sql
-- +goose Up
CREATE TABLE sellers (
    id             BIGSERIAL PRIMARY KEY,
    email          TEXT UNIQUE NOT NULL,
    password_hash  TEXT NOT NULL,
    name           TEXT NOT NULL,
    bio            TEXT DEFAULT '',
    order_email    TEXT DEFAULT '',
    is_active      BOOLEAN DEFAULT false,
    is_admin       BOOLEAN DEFAULT false,
    approval_token TEXT DEFAULT '',
    created_at     TIMESTAMPTZ DEFAULT now(),
    updated_at     TIMESTAMPTZ DEFAULT now()
);

-- +goose Down
DROP TABLE IF EXISTS sellers;
```

### Task 2: Seller entity and SellerStore
Create `internal/models/seller.go` with:

- `Seller` struct with PascalCase fields and `db:` tags matching snake_case columns
- `SellerStore` struct holding a `*pgxpool.Pool`
- Methods:
  - `NewSellerStore(pool *pgxpool.Pool) *SellerStore`
  - `Create(ctx, email, plaintextPassword, name string) (*Seller, error)` — hashes password with bcrypt cost 12, generates a 32-byte random hex approval_token, inserts row, returns Seller
  - `GetByEmail(ctx, email string) (*Seller, error)` — returns nil, nil if not found
  - `GetByID(ctx, id int64) (*Seller, error)` — returns nil, nil if not found
  - `GetByApprovalToken(ctx, token string) (*Seller, error)` — returns nil, nil if not found
  - `Approve(ctx, token string) error` — sets is_active=true, approval_token='' WHERE approval_token=$1
  - `CheckPassword(seller *Seller, plaintext string) bool` — bcrypt.CompareHashAndPassword, returns bool
  - `UpdateProfile(ctx, id int64, name, bio, orderEmail string) error`
  - `ListAll(ctx) ([]Seller, error)` — admin use; ordered by created_at DESC
  - `SetActive(ctx, id int64, active bool) error` — admin approve/reject
  - `AdminExists(ctx) (bool, error)` — returns true if any row has is_admin=true

Files:
- `internal/models/seller.go` — new file

Import `golang.org/x/crypto/bcrypt` (add to go.mod if not present). Use `crypto/rand` for token generation. All DB calls use `pgx/v5` via the pool — consistent with existing ProductStore patterns.
