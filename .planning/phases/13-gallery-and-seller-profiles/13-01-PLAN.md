## Tasks

### Task 1: SellerProfile handler
Read `internal/handlers/public.go` to understand `PublicHandler` struct and existing handler patterns.

Add `SellerProfile(w http.ResponseWriter, r *http.Request)` to `PublicHandler`:
1. Parse `{id}` from the URL path (use `r.PathValue("id")` — Go 1.22+ stdlib routing)
2. Call `h.sellers.GetByID(ctx, id)` — if nil, call `http.NotFound(w, r)` and return
3. Call `h.products.ListBySeller(ctx, id)` — split into available (IsSold=false) and sold (IsSold=true)
4. Execute `seller_profile.html` template with struct containing: `Seller`, `Available []Product`, `PastWork []Product`

Register route in `cmd/server/main.go`: `GET /seller/{id}` → `publicHandler.SellerProfile`

Files:
- `internal/handlers/public.go` — add SellerProfile method
- `cmd/server/main.go` — register GET /seller/{id}

### Task 2: Seller profile template
Create `templates/seller_profile.html`.

Layout matches the existing gallery style:
- Page title: seller name
- Header section: seller name (h1), bio paragraph
- "Available Work" section: product grid using the same card markup as gallery.html
- "Past Work" section: same grid but with sold items; hide section if empty
- Product cards link to `/product/{id}` (same as gallery)
- No auth-gated content on this page

Read `templates/gallery.html` (or equivalent) before creating to reuse the product card markup.

Files:
