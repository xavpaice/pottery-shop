### Task 2: Firing log form template (create + edit)
Create `templates/firings_form.html`:

Used for both New (empty form) and Edit (pre-populated form). The template receives a `FiringLog` struct (empty on create, populated on edit).

Log fields section (above readings):
- Title (text input, required)
- Firing Date (date input, optional)
- Clay Body (text input)
- Glaze Notes (textarea)
- Outcome (text input)
- Notes (textarea)

Readings section:
- `<table>` with header row: Elapsed (min) | Temp (°C) | Gas | Flue | Notes | Remove
- One `<tbody>` with existing readings pre-populated as `<tr>` elements
- A hidden `<tr id="reading-template">` used as the clone source for "Add row"
- "Add Row" button with `onclick`:
  ```js
  function addRow() {
    const t = document.getElementById('reading-template');
    const row = t.cloneNode(true);
    row.removeAttribute('id');
    row.style.display = '';
    const idx = document.querySelectorAll('tbody tr:not(#reading-template)').length;
    row.querySelectorAll('[name]').forEach(el => {
      el.name = el.name.replace('__IDX__', idx);
    });
    document.querySelector('tbody').appendChild(row);
  }
  ```
- Each row input names use pattern `readings[N][elapsed_minutes]`, etc. (template row uses `__IDX__`)
- "Remove" button in each row: `onclick="this.closest('tr').remove()"` (vanilla JS, no library)
- Gas select options: low, medium, high
- Flue select options: 1/4, 1/2, 3/4, open

The entire JS fits in a single `<script>` tag in the template, under 30 lines.

Files:
- `templates/firings_form.html` — new
