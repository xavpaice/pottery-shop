<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Helm values -> ClusterIssuer spec | User-supplied values (acme.email, host) flow into cluster-scoped Kubernetes resources |
| cert-manager -> ACME provider | ClusterIssuer registers with external ACME server and exposes email address |
| ClusterIssuer scope | Cluster-scoped resource; any namespace can reference it for certificate issuance |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | cert-manager-letsencrypt.yaml | mitigate | Default to staging ACME endpoint (`acme-staging-v02`); production requires explicit `ingress.tls.acme.production: true`. Prevents accidental rate-limit burn and ensures operator conscious opt-in (per D-07). |
| T-04-02 | Information Disclosure | cert-manager-letsencrypt.yaml | mitigate | ACME account private key stored in `privateKeySecretRef` Kubernetes Secret with release-scoped name (`clay.fullname`-letsencrypt-key); not logged, not in ConfigMap. RBAC limits access. |
| T-04-03 | Elevation of Privilege | ClusterIssuer resources | accept | ClusterIssuers are cluster-scoped; any namespace can reference them. Acceptable for single-tenant hobby pottery shop. Multi-tenant isolation is documented as future scope (not applicable to current deployment). |
| T-04-04 | Tampering | cert-manager-selfsigned.yaml | mitigate | CA Secret name consistency enforced structurally: both CA Certificate `spec.secretName` and CA ClusterIssuer `spec.ca.secretName` use identical template expression `clay.fullname`-ca-tls. Mismatch is impossible without template source edit. |
| T-04-05 | Denial of Service | Helm hooks | mitigate | `helm.sh/hook-delete-policy: before-hook-creation` on all CRs prevents "already exists" errors during `helm upgrade`, avoiding stuck deployments (per D-05). |
</threat_model>
