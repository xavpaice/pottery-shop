Insert after the "Install CNPG operator" step and before the "Install chart on CMX cluster" step:

```yaml
      # cert-manager is a cluster-scoped operator — install it separately before the clay chart
      # so its CRDs (ClusterIssuer, Certificate) are registered when the chart creates cert-manager resources.
      - name: Add cert-manager Helm repo
        run: helm repo add jetstack https://charts.jetstack.io

      - name: Install cert-manager
        run: |
          helm install cert-manager jetstack/cert-manager \
            --version 1.20.2 \
            --namespace cert-manager \
            --create-namespace \
            --set crds.enabled=true \
            --wait \
            --timeout 3m
```

Per D-16, do NOT change the existing clay chart install step. It already has `--set ingress.enabled=false`, which means no cert-manager CRs will be rendered during the integration test. The cert-manager install is present so that future phases or manual test runs can enable ingress without hitting missing CRD errors.

Key details:
- `--version 1.20.2` -- pinned version per D-15 (released 2025-04-11)
- `--set crds.enabled=true` -- current syntax, NOT the deprecated `installCRDs=true`
- `--namespace cert-manager --create-namespace` -- dedicated namespace
- `--wait --timeout 3m` -- mirrors CNPG pattern
- Helm repo: `jetstack https://charts.jetstack.io` (HTTP repo, matches CNPG pattern)
