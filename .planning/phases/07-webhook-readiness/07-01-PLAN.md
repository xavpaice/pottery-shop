CRITICAL: Each Job gets its own `{{- if }}` / `{{- end }}` guard (NOT a single outer guard for both). This is because the CNPG Job should render when `cloudnative-pg.enabled=true` regardless of cert-manager, and vice versa.

The file MUST contain exactly the following structure:

**Job 1: CNPG webhook-wait**

```yaml
{{- if (index .Values "cloudnative-pg" "enabled") }}
apiVersion: batch/v1
kind: Job
metadata:
  name: {{ include "clay.fullname" . }}-cnpg-webhook-wait
  labels:
    {{- include "clay.labels" . | nindent 4 }}
  annotations:
    "helm.sh/hook": post-install,post-upgrade
    "helm.sh/hook-delete-policy": hook-succeeded,before-hook-creation
    "helm.sh/hook-weight": "-20"
spec:
  backoffLimit: 3
  template:
    metadata:
      name: {{ include "clay.fullname" . }}-cnpg-webhook-wait
      labels:
        {{- include "clay.selectorLabels" . | nindent 8 }}
    spec:
      restartPolicy: Never
      serviceAccountName: {{ include "clay.fullname" . }}-webhook-wait
      containers:
        - name: wait
          image: "alpine/k8s:1.35.3"
          command:
            - sh
            - -c
            - |
              kubectl rollout status deploy/{{ .Release.Name }}-cloudnative-pg \
                -n {{ .Release.Namespace }} --timeout=120s
{{- end }}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Job pod -> Kubernetes API | webhook-wait Job pods make authenticated API calls to query Deployment status; the ServiceAccount token is the trust boundary |
| Container image registry -> cluster | alpine/k8s image pulled from Docker Hub at runtime; image integrity depends on tag pinning |
| Helm release -> RBAC scope | ClusterRole grants cluster-wide read on apps/deployments and apps/replicasets; scope must be minimal |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Elevation of Privilege | ClusterRole rules | mitigate | ClusterRole grants only `get,list,watch` on `apps/deployments` and `apps/replicasets` -- no wildcard verbs, no write access, no other API groups (D-05). Verify with `grep '"*"'` returns empty. |
| T-07-02 | Tampering | alpine/k8s container image | mitigate | Pin to exact version tag `alpine/k8s:1.35.3` -- no `:latest` or mutable tag. Image is pulled once per Job run; tag immutability provides supply-chain integrity at tag level. Full digest pinning deferred to CHART-F01 (post-v1.2). |
| T-07-03 | Information Disclosure | ServiceAccount token in Job pod | accept | The SA has read-only access to deployments/replicasets metadata. No secrets, configmaps, or sensitive resources are accessible. Low risk for a read-only SA with `before-hook-creation` cleanup. |
| T-07-04 | Denial of Service | Job backoffLimit and timeout | mitigate | `backoffLimit: 3` prevents infinite retry loops. `--timeout=120s` on kubectl prevents indefinite hang. Job failure causes Helm to report error and stop release -- does not leave orphaned resources. |
| T-07-05 | Spoofing | Operator deployment name construction | mitigate | Deployment names use `{{ .Release.Name }}-cloudnative-pg` and `{{ .Release.Name }}-cert-manager-webhook` -- derived from Helm release name template, verified against `helm template` output. No user-controlled input beyond Release.Name which Helm sanitizes. |
| T-07-06 | Elevation of Privilege | RBAC hook ordering | mitigate | RBAC at weight -25 (pre-install), Jobs at weight -20 (post-install). Helm enforces weight ordering. If RBAC fails to create, Jobs cannot run (no SA = pod scheduling failure, visible error). |
</threat_model>
