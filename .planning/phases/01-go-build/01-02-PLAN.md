**Step 3: Replace the database open block**

Remove (lines ~36-45 in current file):
```go
// Database
db, err := sql.Open("sqlite3", dbPath+"?_foreign_keys=on")
if err != nil {
    log.Fatalf("Failed to open database: %v", err)
}
defer db.Close()

store := models.NewProductStore(db)
if err := store.Init(); err != nil {
    log.Fatalf("Failed to initialise database: %v", err)
}
```

Replace with:
```go
// Database
pool, err := pgxpool.New(context.Background(), databaseURL)
if err != nil {
    log.Fatalf("Failed to create connection pool: %v", err)
}
defer pool.Close()

db := stdlib.OpenDBFromPool(pool)
defer db.Close()

// Run migrations
goose.SetBaseFS(migrations.FS)
if err := goose.SetDialect("postgres"); err != nil {
    log.Fatalf("Failed to set goose dialect: %v", err)
}
if err := goose.Up(db, "."); err != nil {
    log.Fatalf("Failed to run migrations: %v", err)
}

store := models.NewProductStore(db)
```

Key points per decisions:
- D-01/D-02: `pgxpool.New` creates the pool, `stdlib.OpenDBFromPool` wraps it as `*sql.DB`
- D-03: ProductStore receives `*sql.DB`, not the native pool
- D-10: `goose.Up(db, ".")` runs after pool is open, before HTTP server starts. Fatal on failure. NOTE: D-10 text says `goose.Up(db, "migrations")` but this is superseded by RESEARCH.md Pitfall 4 — when using the `internal/migrations` package pattern where `migrations.FS` embeds `*.sql` at the package root via `//go:embed *.sql`, the correct `goose.Up` path is `"."` (root of the embedded FS), not `"migrations"`. Using `"migrations"` would silently find zero migration files and create no tables.
- The `store.Init()` call is removed entirely (D-07 — Init was deleted in Plan 01)

The `envOr` helper function at the bottom of the file is kept (still used for PORT, BASE_URL, ADMIN_USER, etc.).

**Dockerfile changes (D-13, D-14, D-15):**

Rewrite the entire Dockerfile. The new content:

```dockerfile
# Build stage — pure Go, no CGO
FROM golang:1.26-alpine AS builder

WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download

COPY . .
RUN CGO_ENABLED=0 go build -o clay-server ./cmd/server

# Runtime stage
FROM alpine:3.20

RUN apk add --no-cache ca-certificates

WORKDIR /app

COPY --from=builder /app/clay-server .
COPY --from=builder /app/templates ./templates
COPY --from=builder /app/static ./static

# Data directories — mount volumes here
RUN mkdir -p /data/uploads/thumbnails

ENV PORT=8080
ENV UPLOAD_DIR=/data/uploads

EXPOSE 8080

CMD ["./clay-server"]
```

- D-13: Entire `tonistiigi/xx:1.6.1` stage and `COPY --from=xx / /` are gone
- D-13: `ARG TARGETPLATFORM`, `apk add clang lld`, `xx-apk add gcc musl-dev` are gone
- D-14: Build line is now `CGO_ENABLED=0 go build -o clay-server ./cmd/server` (not `xx-go`, not `xx-verify`)
- D-15: Runtime `apk add` no longer includes `sqlite-libs`; keeps `ca-certificates`
- D-12: `ENV DB_PATH=/data/clay.db` line is removed
