## Tasks

### Task 1: Seller dashboard — scoped product list and create
Read `internal/handlers/auth.go`.

Add to `AuthHandler` (or `DashboardHandler`):
- `DashboardProducts(w, r)` — GET /dashboard/products; calls `ProductStore.ListBySeller(ctx, session.SellerID)`; renders dashboard products list template
- `DashboardNewProduct(w, r)` — GET /dashboard/products/new; renders the new product form
- `DashboardCreateProduct(w, r)` — POST /dashboard/products/create; reads form fields, calls `ProductStore.Create(ctx, ..., session.SellerID)`; redirects to /dashboard/products
- `DashboardEditProduct`, `DashboardUpdateProduct`, `DashboardDeleteProduct`, `DashboardToggleSold` — mirror AdminHandler equivalents but enforce `product.SellerID == session.SellerID` before any mutation (return 403 otherwise)

Register all routes in `cmd/server/main.go` under `/dashboard/products/*`, guarded by `requireSeller`.

Files:
- `internal/handlers/auth.go` — add dashboard product methods
- `cmd/server/main.go` — register dashboard product routes

### Task 2: Admin dashboard — all products with seller column
Read `internal/handlers/admin.go`.

Update `AdminHandler.Dashboard()` to call `ProductStore.ListAllWithSeller(ctx)` instead of the plain `ListAll`. The returned `Product` structs now carry `SellerName` from the JOIN.

Update the admin dashboard template to add a "Seller" column in the products table.

Files:
- `internal/handlers/admin.go` — update Dashboard() to use ListAllWithSeller
- `templates/admin/dashboard.html` — add Seller column to products table (read template before editing)

### Task 3: Order email routing to seller
Read `internal/handlers/public.go` to find the `PlaceOrder` handler and the SMTP send logic.

After fetching the product for an order, fetch the seller:
```go
seller, err := h.sellers.GetByID(ctx, product.SellerID)
```
Determine the recipient email:
```go
toEmail := h.config.OrderEmail // global fallback
if seller != nil && seller.OrderEmail != "" {
    toEmail = seller.OrderEmail
}
```
Pass `toEmail` to the existing email-send helper instead of always using `h.config.OrderEmail`.

Files:
- `internal/handlers/public.go` — update PlaceOrder to route email to seller.OrderEmail or fallback
